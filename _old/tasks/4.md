# テーマ作成画面のステップウィザード化

## Context

現在のテーマ作成画面(`apps/mobile/app/theme/new.tsx`)は3つの入力(テーマ・相手・目的)を1画面に表示するフォーム。これをカードスライド式の3ステップウィザードに変更し、UXを改善する。

- Step 1: テーマを入力 + tip候補 → 次へ
- Step 2: 相手を選択(ペルソナグリッド) → 次へ
- Step 3: 目的を入力 + tip候補 → テーマを作成する

## Phase 1: Backend — ペルソナデータモデル & API

### 1.1 Core型定義 — `packages/core/src/persona.ts` (新規)

```typescript
PersonaSchema = { id, name(max40), description(max120), emoji(max4) }
ListPersonasResponseSchema = { personas: PersonaSchema[] }
```

`packages/core/src/index.ts` からre-export。

### 1.2 ThemeInputSchema拡張 — `packages/core/src/practice.ts`

`personaId: z.string().trim().min(1).optional()` を追加。既存データとの後方互換性あり。

## Phase 2: Mobile API層

### 2.1 APIクライアント — `apps/mobile/src/lib/api.ts`

```typescript
export async function fetchPersonas(): Promise<Persona[]> {
  const payload = await fetchJson<{ personas: Persona[] }>(`${API_BASE_URL}/v1/personas`);
  return payload.personas;
}
```

### 2.2 フォームヘルパー — `apps/mobile/src/lib/theme-form-helpers.ts`

- `ThemeFormState` に `personaId?: string` を追加
- `audience` はペルソナ名から自動導出
- ステップ単位のバリデーション関数を追加:
  - `validateThemeStep()` — theme のみ
  - `validateGoalStep()` — goal のみ

## Phase 3: ステップウィザードUI

### 3.1 新規コンポーネント — `apps/mobile/src/components/theme-wizard/`

| ファイル | 役割 |
|---------|------|
| `step-indicator.tsx` | ステップ進捗ドット (1/2/3) |
| `step-theme.tsx` | Step1: テーマ入力 + tipチップ |
| `step-persona.tsx` | Step2: ペルソナ3列グリッド選択 |
| `step-goal.tsx` | Step3: 目的入力 + tipチップ |
| `tip-chips.tsx` | タップで入力されるヒントチップ(共通) |
| `persona-card.tsx` | 個別ペルソナカード(emoji + name + description) |

### 3.2 Tipデータ — `apps/mobile/src/lib/theme-wizard-tips.ts` (新規)

```typescript
export const THEME_TIPS = ["今日の天気について", "最近見たアニメ", "週末の予定", "おすすめの本"];
export const GOAL_TIPS = ["相手に興味を持ってもらう", "共感してもらう", "情報を正確に伝える"];
```

### 3.3 ウィザードオーケストレータ — `apps/mobile/app/theme/new.tsx` (書き換え)

- `currentStep` state (0, 1, 2) で管理
- `react-native-reanimated` の `useSharedValue` + `withTiming` でX軸スライドアニメーション
- 3ステップを横並び配置、`overflow: hidden` で1つだけ表示
- ペルソナはマウント時にprefetch(Step1の間にロード完了)
- ペルソナ選択時に `audience = persona.name` を自動セット(LLMプロンプト互換)
- 各ステップで部分バリデーション、最終submitで全体バリデーション
- 戻るボタンでステップを戻れる

### アニメーション実装

```typescript
const translateX = useSharedValue(0);
const { width: screenWidth } = useWindowDimensions();

function goToStep(step: number) {
  setCurrentStep(step);
  translateX.value = withTiming(-step * screenWidth, { duration: 300 });
}
```

## 注意事項

- `audience` フィールドはペルソナ名をそのまま文字列として渡す（LLMプロンプト互換のため）
- `personaId` はoptionalで後方互換性を維持
- ペルソナはグローバルデータ（owner_keyなし）
- `useWindowDimensions()` で画面幅取得（回転対応）

## 検証方法

1. `pnpm -r build` で全パッケージビルド通過
2. `pnpm -r test` で全テスト通過
3. Expo dev serverでウィザードUIを実機/シミュレータ確認
   - Step間のスライドアニメーション
   - tipタップで入力反映
   - ペルソナ3列グリッド表示・選択
   - テーマ作成の完走(API送信→詳細画面遷移)
4. 既存テーマの表示が壊れないこと(後方互換)
