# kotoba-gym Phase 0b 仕様書 (音声ループ検証 / Gemini ネイティブ音声版)

## 0. 前提

Phase 0a(AI頭脳の検証)が完了し、テキストCLIで PM ペルソナとの会話・チェックリスト判定・フィードバック生成が成立していることが前提。本フェーズは **Phase 0a のコアロジック(`session.ts`)を再利用** しつつ、音声対話に拡張する。

## 1. Phase 0b のゴール

**検証対象: 音声対話が「生きた会話」として成立するか + 音響情報を評価に活かせるか**

- 音声入出力のレイテンシが会話として自然か
- 沈黙で「あれ?」とならない会話テンポが出せるか
- **Gemini の音響理解を使って、テキストでは取れないフィードバック(フィラー・間・自信のなさ)が生成できるか**

言語化アプリの差別化は、ここの音響情報を評価に組み込めるかにかかっている。単なる UI の話ではなく、**サービスの中核機能の検証**。

## 2. アーキテクチャの方針(案B)

従来の「STT → LLM → TTS」パイプラインを廃し、**音声を Gemini 3 Flash に直接送る**構成を採用。

- サーバーサイド STT は使わない
- Web Speech API は **ユーザー体験上の「聞いてます」表示だけ** に使う(裏処理には使わない)
- Gemini 3 Flash が音声を直接理解して返答テキストを生成
- 返答テキストを Gemini 3.1 Flash TTS(または Google Cloud TTS)で音声化

## 4. 技術スタック

### フロントエンド

- **フレームワーク**: Vite + 素の TypeScript(React 不要、軽く済ます)
- **音声録音**: `MediaRecorder API`(audio/webm; codecs=opus で録音 → blob 送信)
- **字幕表示用 STT**: `Web Speech API`(**表示目的のみ**、録音品質の保険にもなる)
- **TTS再生**: `<audio>` ストリーミング再生 or Web Audio API
- **通信**: Server-Sent Events (SSE) でテキストストリーム、音声は別エンドポイントで byte stream

### バックエンド

- **フレームワーク**: Hono + `@hono/node-server`
- **LLM**: **Gemini 3 Flash**(音声入力対応、Persona と Judge 両方で使用)
  - ※ Gemini 3 Pro Preview は 2026年3月9日に deprecate 済み、現行は 3.1 系
  - モデル名は環境変数で切り替え可能に
- **TTS**: **Gemini 3.1 Flash TTS** を第一候補、代替に Google Cloud TTS
  - Gemini TTS は audio tags でスタイル・ペース制御可能(将来のキャラ付けに有利)
  - 日本語クオリティを実測した上で選ぶ

## 5. 会話ループの1ターン

1. ユーザーがマイクボタンをタップ → 録音開始
2. **並行して2つ動く**:
   - a. `MediaRecorder` が音声を蓄積
   - b. `Web Speech API` が interim transcript を画面表示(聞いてます感)
3. ユーザーがもう一度タップ → 録音終了、audio blob を POST `/reply` へ送信
4. サーバー:
   - a. **Persona 呼び出し**(Gemini 3 Flash):音声 + 会話履歴 → 返答テキストをストリーミング
   - b. **Judge 呼び出し**(Gemini 3 Flash):音声 + 現状のチェックリスト → 更新 JSON(並列・fire-and-forget)
5. Persona のトークンを SSE でフロントに流す
6. フロントは **文単位(句点区切り)で TTS を呼び**、音声を順次再生
7. 最後の文まで再生 → ユーザー発話待ちに戻る

### Persona と Judge を分けて両方に音声を渡す理由

- Persona は返答生成に集中(音響情報は返答の自然さのためにも効く)
- Judge は裏で音響特徴もチェックリスト評価に組み込める
- 分離しておくと、後で Judge だけモデル変更や最適化がしやすい

コストは倍になるが、音声入力は$1/1M input tokensで、1分の音声でも知れた量。Phase 0b の規模では無視できる。

## 6. 音響情報の活用

**ここが案A(従来STT)との最大の差**。Judge と Feedback で以下を評価対象にする。

### Judge の評価軸(案)

従来のチェックリスト充足度に加えて、音響面の観察を出力:

```ts
export const JudgeResultSchema = z.object({
  updates: z.array(z.object({
    id: z.string(),
    newState: z.enum(["empty", "partial", "filled"]),
    reason: z.string(),
  })),
  acousticObservations: z.array(z.object({
    type: z.enum(["filler", "long_pause", "unclear_pronunciation", "confident", "hesitant"]),
    context: z.string(), // 「成果指標の話の時」など
    note: z.string(),
  })).optional(),
});
```

### Feedback への反映

セッション終了時のフィードバックで、音響観察を自然に混ぜる:

- 「成果指標のところで『えーっと』が続きました。ここは数字を先に用意しておくと流れが変わります」
- 「提案の切り出しは歯切れ良かったです。同じテンションで課題説明も押し切れると伝わります」

テキストSTTでは絶対に出せない、**体感で刺さるフィードバック**。サービスの差別化ポイントそのもの。

## 7. レイテンシ目標と計測

### 目標値

| 区間                         | 目標  | 許容上限 |
| ---------------------------- | ----- | -------- |
| 録音終了 → 1音節目が聞こえる | 1.5秒 | 3.0秒    |
| 音声アップロード完了         | 0.3秒 | 0.8秒    |
| LLM 初トークン到達 (TTFT)    | 0.8秒 | 1.5秒    |
| TTS 初chunk到達              | 0.4秒 | 1.0秒    |

### 計測方法

`performance.now()` で各区切りを記録、セッション終了時にコンソール出力:

```ts
const marks = {
  recordingEnd: 0,
  uploadComplete: 0,
  llmFirstToken: 0,
  ttsFirstChunk: 0,
  firstAudioPlay: 0,
};
```

**数字は目安、最終判断は定性的に**。「会話している」と自分が感じるかが全て。

## 8. レイテンシ最適化テクニック

目標が出なかった時の手札:

### a. 並列化

- Judge と Persona を並列発火、Persona ストリームを止めない
- Judge の結果はターン終了後にマージ

### b. ストリーミングTTS

- Gemini からトークンを受け取りながら、**句点/読点区切りで即 TTS に流す**
- 1文目の音声が返ったら即再生、2文目以降は queue に積む

### c. PM返答を短くする

- ペルソナプロンプトに「返答は2〜3文以内、質問は1つ」と明記
- 短さは音声時のレイテンシ最適化そのもの

### d. 音声圧縮

- Opus 16kbps でも音声認識には十分、アップロード時間短縮

### e. フィラー挿入(最終手段)

- 初トークン待ちが長い時、先に固定音声で「えーっと」を流して時間稼ぎ
- 本返答が来たらクロスフェード
- 実装重いので、ここまでやるのは他で詰んでから

### エンドポイント

- `POST /session` → セッション作成、`sessionId` 返却
- `POST /reply` (multipart/form-data) → `audio` blob + `sessionId` → SSE で `data: { type: "token", text }` → 完了時 `data: { type: "done", checklist, acousticObservations, isEnded }`
- `POST /tts` → `{ text }` → audio/mpeg ストリーム
- `POST /finalize` → `{ sessionId }` → Feedback JSON

セッション状態はインメモリ保持(Phase 0b では十分)。

## TTSのサンプルコード

// To run this code you need to install the following dependencies:
// npm install @google/genai mime
// npm install -D @types/node

import {
  GoogleGenAI,
} from '@google/genai';
import mime from 'mime';
import { writeFile } from 'fs';

function saveBinaryFile(fileName: string, content: Buffer) {
  writeFile(fileName, content, 'utf8', (err) => {
    if (err) {
      console.error(`Error writing file ${fileName}:`, err);
      return;
    }
    console.log(`File ${fileName} saved to file system.`);
  });
}

async function main() {
  const ai = new GoogleGenAI({
    apiKey: process.env['GEMINI_API_KEY'],
  });
  const config = {
    temperature: 1,
    responseModalities: [
        'audio',
    ],
    speechConfig: {
      voiceConfig: {
        prebuiltVoiceConfig: {
          voiceName: 'Zephyr',
        }
      }
    },
  };
  const model = 'gemini-3.1-flash-tts-preview';
  const contents = [
    {
      role: 'user',
      parts: [
        {
          text: `## Transcript:
`,
        },
      ],
    },
  ];

  const response = await ai.models.generateContentStream({
    model,
    config,
    contents,
  });
  let fileIndex = 0;
  for await (const chunk of response) {
    if (!chunk.candidates || !chunk.candidates[0].content || !chunk.candidates[0].content.parts) {
      continue;
    }
    if (chunk.candidates?.[0]?.content?.parts?.[0]?.inlineData) {
      const fileName = `ENTER_FILE_NAME_${fileIndex++}`;
      const inlineData = chunk.candidates[0].content.parts[0].inlineData;
      let fileExtension = mime.getExtension(inlineData.mimeType || '');
      let buffer = Buffer.from(inlineData.data || '', 'base64');
      if (!fileExtension) {
        fileExtension = 'wav';
        buffer = convertToWav(inlineData.data || '', inlineData.mimeType || '');
      }
      saveBinaryFile(`${fileName}.${fileExtension}`, buffer);
    }
    else {
      console.log(chunk.text);
    }
  }
}

main();

interface WavConversionOptions {
  numChannels : number,
  sampleRate: number,
  bitsPerSample: number
}

function convertToWav(rawData: string, mimeType: string) {
  const options = parseMimeType(mimeType)
  const wavHeader = createWavHeader(rawData.length, options);
  const buffer = Buffer.from(rawData, 'base64');

  return Buffer.concat([wavHeader, buffer]);
}

function parseMimeType(mimeType : string) {
  const [fileType, ...params] = mimeType.split(';').map(s => s.trim());
  const [_, format] = fileType.split('/');

  const options : Partial<WavConversionOptions> = {
    numChannels: 1,
  };

  if (format && format.startsWith('L')) {
    const bits = parseInt(format.slice(1), 10);
    if (!isNaN(bits)) {
      options.bitsPerSample = bits;
    }
  }

  for (const param of params) {
    const [key, value] = param.split('=').map(s => s.trim());
    if (key === 'rate') {
      options.sampleRate = parseInt(value, 10);
    }
  }

  return options as WavConversionOptions;
}

function createWavHeader(dataLength: number, options: WavConversionOptions) {
  const {
    numChannels,
    sampleRate,
    bitsPerSample,
  } = options;

  // http://soundfile.sapp.org/doc/WaveFormat

  const byteRate = sampleRate * numChannels * bitsPerSample / 8;
  const blockAlign = numChannels * bitsPerSample / 8;
  const buffer = Buffer.alloc(44);

  buffer.write('RIFF', 0);                      // ChunkID
  buffer.writeUInt32LE(36 + dataLength, 4);     // ChunkSize
  buffer.write('WAVE', 8);                      // Format
  buffer.write('fmt ', 12);                     // Subchunk1ID
  buffer.writeUInt32LE(16, 16);                 // Subchunk1Size (PCM)
  buffer.writeUInt16LE(1, 20);                  // AudioFormat (1 = PCM)
  buffer.writeUInt16LE(numChannels, 22);        // NumChannels
  buffer.writeUInt32LE(sampleRate, 24);         // SampleRate
  buffer.writeUInt32LE(byteRate, 28);           // ByteRate
  buffer.writeUInt16LE(blockAlign, 32);         // BlockAlign
  buffer.writeUInt16LE(bitsPerSample, 34);      // BitsPerSample
  buffer.write('data', 36);                     // Subchunk2ID
  buffer.writeUInt32LE(dataLength, 40);         // Subchunk2Size

  return buffer;
}
