# 練習モデル変更: 「2回で完結するセッション」→「テーマに紐づく繰り返し練習」

## Context

現在のモデルは1セッション = 最大2回の回答（attempt）で、1回目の後に「中断→再開(CONTINUE)」という体験になっている。これを変更して:

- **中断の概念をなくす** — 毎回の練習が独立した1セッションになる
- **前回との比較は残す** — 同じテーマの直前セッションと比較する
- **何度でも繰り返し練習** — 2回制限を撤廃
- **テーマ詳細に練習履歴** — テーマ一覧 → 詳細画面からセッションリストを確認
- **ホームのCONTINUEカード削除** — 「最近のテーマ」で代替

## 方針

セッション = 1回の練習（evaluation直付け）にフラット化する。`attempts` 配列を廃止し、`evaluation: AttemptEvaluation | null` をセッション直下に持つ。前回比較はサーバーが同テーマの直前セッションを自動で取得して行う。

## Phase 1: Core スキーマ変更

**`packages/core/src/practice.ts`**

- `PracticeSessionAttemptSchema` を削除
- `PracticeSessionRecordSchema` を変更:
  ```
  attempts: array → 削除
  evaluation: AttemptEvaluationSchema.nullable().default(null)  ← 追加
  recordedAt: z.string().nullable().default(null)               ← 追加
  ```
- `PreviousAttemptPayloadSchema` から `attemptNumber` を削除
- 対応する type export を更新

## Phase 2: サーバー変更

### DB マイグレーション
**`apps/server/migrations/0002_add_theme_id.sql`** (新規)
```sql
ALTER TABLE sessions ADD COLUMN theme_id TEXT NOT NULL DEFAULT '';
CREATE INDEX idx_sessions_owner_theme ON sessions(owner_key, theme_id, updated_at DESC);
```

### Repository
**`apps/server/src/repositories/app-repository.ts`**

- `AppRepository` interface に追加:
  - `listSessionsByTheme(params: { ownerKey, themeId }): Promise<PracticeSessionRecord[]>`
- `D1AppRepository`:
  - `saveSession`: INSERT に `theme_id` カラム追加
  - `listSessionsByTheme`: `WHERE owner_key = ? AND theme_id = ?` で取得
  - `parseSession`: レガシー形式（`attempts` 配列）→ 新形式への変換を追加
- `InMemoryAppRepository`: 同様に実装
- `listSessions` に `?themeId` フィルタ対応（既存 route で使う）

### セッション操作
**`apps/server/src/lib/session-record.ts`**

- `createPracticeSessionRecord`: `attempts: []` → `evaluation: null, recordedAt: null`
- `upsertPracticeSessionAttempt` → `setSessionEvaluation` にリネーム（evaluation を直接セット）
- `toPreviousAttemptPayload`: `attemptNumber` 引数を削除

### ルート
**`apps/server/src/v1/routes.ts`**

- `POST /v1/evaluations`:
  - `attempts.length >= 2` チェックを削除
  - `expectedAttemptNumber` / `attemptNumber` 整合チェックを削除
  - 前回の evaluation を `repository.listSessionsByTheme` で同テーマの直前完了セッションから取得
  - `setSessionEvaluation` で session を更新
- `GET /v1/sessions`: `?themeId` クエリパラメータ対応追加

### ルートヘルパー
**`apps/server/src/v1/route-helpers.ts`**

- `EvaluationFieldsSchema` から `attemptNumber` を削除（または無視）

### 評価ヘルパー
**`apps/server/src/v1/evaluation.ts`** / **`evaluation-helpers.ts`**

- `attemptNumber` パラメータを削除 or 固定値化（LLMプロンプトへの影響は最小）

   - ホーム画面にCONTINUEカードが出ないことを確認
